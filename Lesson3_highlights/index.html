Lesson 3 Highlights

Justin Morris

Highlight #1 Version Control Systems

Version control systems were easily the coolest thing learned in this lesson. They make data and file management much easier for both average users and expert commercial users. There are many different companies that host these systems which all have repositories for saving different versions of projects.Version Control systems also fix the issue of shared files, multiple people can edit the files without affecting or losing the previous version. This makes it easier to compare ideas and fix errors. The systems automatically keep track of when a work is edited, saving the user time as well.

Highlight #2 GIT

GIT is what many web developers use as their version control system. GIT is an online system that works with most IDE programs to help manage the data. I find it very interesting that such big companies with big data and files still use programs like GIT. Private users pay for the service, but they allow anyone to use it for public display at any time (to a certain extent). You can also have it setup to work with your IDE, so one can enter a command and then have it update the local repository and the remote repositories instantly.

Highlight #3 Local and Remote Repositories

I find the use of local and remote repositories to be quite ingenious. this obviously makes user more confident in backing up their data. Local repositories store things on your own local device while remote repositories are at a completely different location, and use some type of cloud server.

Highlight #4 Using the Terminal

Cloud9 is a great IDE and has a built in terminal (based on a Linux Command Prompt). The terminal makes it easy to use commands from Git hub to save different versions of my project. Git stores every version of almost any project and has its own unique commands. This terminal is also used for linking other services to Cloud9's code editor.

Highlight #5 The Sandbox

Git hub makes itself super easy to learn as well. They provide a location called the sand box, this is where one can go to practice and try out the different commands. This sandbox won't affect major projects, nor will it affect the status of one's repositories.
